# 信息安全部职责

信息安全部负责安全策略制定、安全事件响应、恶意程序处置、账号与权限风险排查、漏洞管理、日志审计、数据安全治理和安全培训。凡是涉及病毒木马、钓鱼攻击、账号被盗、异常外联、敏感数据泄露、权限滥用或安全合规要求解释的事项，都应优先由信息安全部主导。

典型受理事项包括：办公电脑中毒事件处置、勒索软件预警、钓鱼邮件排查、终端异常流量分析、服务器漏洞通报、权限过宽整改、数据导出风险评估、外发文档加密要求确认以及安全事件复盘。信息安全部在处置过程中会要求相关设备先隔离，再进行日志保全、影响面排查和修复建议输出。

信息安全部不负责日常报销、不负责合同商务谈判、不负责供应商下单，也不负责普通设备采购流程。若事项只是“电脑开不了机”或“打印机不能用”，应由 IT支持部处理；若事项是“安全产品采购”，采购部负责商务流程，但技术与风险评估仍需要信息安全部参与；若涉及法律通知或监管沟通，法务部需要同步介入。

当业务部门提出“办公电脑中毒该找谁”“发现疑似钓鱼邮件由哪个部门处理”“账号异常登录谁来排查”时，应明确路由到信息安全部，由其统一指挥并安排 IT支持部等协同资源。
